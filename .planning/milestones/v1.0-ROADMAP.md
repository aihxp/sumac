# Milestone v1.0: Product-Preserving Greenfield Core Rewrite

**Status:** Completed 2026-04-05
**Phases:** 1-6
**Total Plans:** 10

## Overview

This milestone preserved the shipped Sumac product while rebuilding the
maintained onboarding and reconciliation path around a cleaner internal
core/app seam. The work locked down the golden-path contract baseline, added
parity proof, introduced the new seam, then migrated `status`, `sync`, `add`,
and `setup` in dependency order without breaking CLI behavior, JSON contracts,
or generated artifacts.

## Milestone Summary

### Key Accomplishments

- published `docs/GOLDEN_PATH_CONTRACT.md` as the rewrite source of truth
- added named Rust and shell rewrite parity checks for the golden path
- introduced `src/app/` and moved the maintained workflow behind a typed app
  seam
- migrated `status`, `sync`, `add`, and `setup` into dedicated service modules
- extracted shared onboarding into `src/app/onboarding.rs`
- reduced `src/app/golden_path.rs` to a thin dispatcher across the migrated
  command family

### Issues Resolved

- removed the need for golden-path orchestration to live in `src/main.rs`
- replaced one-off migration logic with a repeatable per-command service pattern
- added direct legacy-vs-core parity proof for the maintained lifecycle

### Deferred

- keep `SXMC_GOLDEN_PATH_ROUTE=legacy` until a stable release cycle ships
  without needing the rollback seam
- broader subsystem migration stays for later milestones
