# Plan 06-01: Rebuild `setup` over Shared Onboarding and Reconciliation Services

## Goal

Move `setup` into a dedicated service module that reuses the Phase 5
onboarding service and removes the final command-specific implementation from
`src/app/golden_path.rs`.

## Tasks

1. Add `src/app/setup.rs`.
2. Move `SetupRequest` and setup execution into `SetupService`.
3. Rewire `main.rs` and `GoldenPathApp` to use the dedicated setup service.
4. Add setup-specific legacy-route and core-vs-legacy parity tests.

## Done When

- `setup` no longer lives as command logic in `src/app/golden_path.rs`
- `GoldenPathApp` is reduced to dispatch across all four migrated commands
- setup parity proof exists for default/core versus explicit legacy routing
