# Plan 02-02: Define Typed Contracts and Explicit Adapters

## Goal

Replace ad hoc golden-path orchestration with typed internal contracts and
explicit adapter boundaries around existing infrastructure-heavy behavior.

## Tasks

1. Define typed request/result structs for the migrated command family.
2. Create explicit adapters for:
   - install-scope/path resolution
   - host detection
   - profile inspection/readiness
   - artifact resolution/materialization
   - status collection
   - sync collection
3. Reuse current helpers through those adapters instead of letting `main.rs`
   orchestrate directly.

## Done When

- command-family requests/results are explicit types
- the app seam depends on adapters instead of sprawling direct orchestration
- later phases can replace adapters slice by slice without changing the seam
