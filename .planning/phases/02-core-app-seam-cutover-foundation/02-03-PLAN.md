# Plan 02-03: Add Cutover Routing and Shim Retirement Rules

## Goal

Make the new seam safe to ship incrementally by adding explicit routing,
rollback control, and retirement criteria for the legacy shim path.

## Tasks

1. Introduce a golden-path route selector with a rollback-safe legacy mode.
2. Wire `setup`, `add`, `status`, and `sync` through the route selector.
3. Record shim-retirement rules in the phase summary/verification.
4. Confirm parity tests cover the routed path.

## Done When

- the route is explicit
- stable releases can fall back safely if needed
- shim retirement expectations are written down for later phases
